Create a Repository

Your repository is the project folder that will hold all your website files.

← Back to Getting Started

What's a Repository?

A repository (or "repo") is just a folder on GitHub that holds all the files for a single project. For your design system site, you'll create one repository that contains all of your HTML pages, your CSS file, and any images.

Step-by-Step: Create Your Repo

  1. Log in to github.com.
  2. In the top-right corner, click the + button and select "New repository".
  3. Name your repository. Use a short, lowercase name with hyphens instead of spaces. For example: design-system or dsn326-project.
    Why this matters: The repo name becomes part of your URL. If your username is janedoe and your repo is called design-system, your site will live at janedoe.github.io/design-system.
  4. Add a description (optional but helpful). Something like "DSN 326 Design System Project" works great.
  5. Set it to Public. GitHub Pages only works with public repositories on the free plan. Make sure the "Public" option is selected.
    Important: If you set your repo to "Private," GitHub Pages won't work unless you have a paid GitHub plan. Always choose Public.
  6. Check "Add a README file." This creates a small starter file in your repo. It's not required for your website, but it makes the repo easier to work with right away.
  7. Leave everything else as-is (no .gitignore, no license needed).
  8. Click "Create repository".
Done! You now have an empty repository ready to hold your website files. You should see your new repo page with just a README.md file in it.

What About the username.github.io Repo?

You may have read that you need a repository named yourusername.github.io. That's only needed for a personal homepage site. For this class, you're creating a project site, which can be any repo name you want. Both types work exactly the same way with GitHub Pages.

Quick Checklist

  • I created a new repository
  • The repo name is lowercase with hyphens (no spaces)
  • The repo is set to Public
  • I checked "Add a README file"
