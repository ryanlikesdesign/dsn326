DSN 326 — Step 08

Previewing Your Work

How to see your website on your own computer before uploading it to GitHub.

← Back to Getting Started

Why Preview Locally?

Every time you upload files to GitHub and wait for Pages to deploy, it takes a minute or two. That adds up fast when you're making lots of small changes. Instead, you can preview your site on your own computer before uploading — so you only push files to GitHub when you're happy with how they look.

Method 1: Just Open the File (No Software Needed)

This is the absolute simplest way to preview your work. No downloads, no setup.

  1. Find your index.html file on your computer (in Finder on Mac, or File Explorer on Windows).
  2. Double-click it. It will open in your default web browser.
  3. You'll see your page rendered just like a real website. The URL bar will show a local file path instead of a web address — that's normal.
  4. When you make changes to your HTML or CSS, save the file in your text editor, then go back to the browser and refresh the page (F5 or Cmd+R / Ctrl+R).
This works great for: Quick checks, simple edits, or if you don't want to install anything. The only downside is you have to manually refresh the browser every time you change something.

Method 2: VS Code + Live Server (Recommended)

Visual Studio Code (VS Code) is a free code editor made by Microsoft. It's the most popular editor for web development, and with one extension called Live Server, it can auto-refresh your browser every time you save a file — so you see changes instantly.

Step 1: Download VS Code

  1. Go to code.visualstudio.com.
  2. Click the big Download button. It will detect whether you're on Mac or Windows automatically.
  3. Mac: Open the downloaded .zip file. Drag the "Visual Studio Code" app into your Applications folder. Then open it from Applications.
  4. Windows: Run the downloaded .exe installer. Click through the setup wizard (the defaults are fine). Check the box that says "Add to PATH" if you see it.

Step 2: Open Your Project Folder

  1. Open VS Code.
  2. Go to File → Open Folder (on Mac: File → Open…).
  3. Navigate to the folder on your computer that contains your project files (index.html, style.css, etc.) and select it.
  4. You'll see your files listed in the sidebar on the left. Click any file to open it in the editor.
Tip: Always open the folder, not just a single file. This way VS Code sees all your project files and the file paths in your HTML will work correctly.

Step 3: Install the Live Server Extension

  1. In VS Code, click the Extensions icon in the left sidebar. It looks like four small squares (or press Ctrl+Shift+X on Windows / Cmd+Shift+X on Mac).
  2. In the search bar at the top, type Live Server.
  3. Look for the one by Ritwick Dey — it should be the top result and has millions of downloads.
  4. Click the blue Install button.
  5. Wait a few seconds for it to install. That's it — no restart needed.
You only need to do this once. The extension stays installed in VS Code permanently. You won't need to install it again for future projects.

Step 4: Launch Live Server

  1. In the VS Code sidebar, click on your index.html file to open it.
  2. Now do one of the following:
    • Option A: Right-click anywhere in the index.html file and select "Open with Live Server".
    • Option B: Look at the very bottom of the VS Code window (the blue status bar). You should see a "Go Live" button on the right side. Click it.
  3. A browser window will automatically open showing your website at an address like http://127.0.0.1:5500.

Step 5: Edit and Watch It Update

Now here's the best part: make a change in VS Code (like changing a heading or a color in your CSS), then save the file (Ctrl+S / Cmd+S). Your browser will automatically refresh and show the change instantly. No clicking refresh, no waiting.

The workflow:
1. Edit code in VS Code
2. Press Ctrl+S / Cmd+S to save
3. Browser updates automatically
4. Repeat until you're happy
5. Upload finished files to GitHub

To Stop Live Server

Click the "Port: 5500" button in the VS Code status bar (bottom-right), or close VS Code. The local preview will stop.

Recommended Workflow for This Class

Here's the workflow that will save you the most time:

1. Build & preview locally.
Keep your project folder on your computer. Edit files in VS Code, preview with Live Server. Make all your changes here first.
2. Upload to GitHub when ready.
Once your page looks good locally, go to your GitHub repo, click "Add file → Upload files," and drag your updated files in. Commit the changes.
3. Verify on GitHub Pages.
Wait a minute or two, then visit your github.io URL to make sure everything looks right on the live site. Do a hard refresh (Ctrl+Shift+R / Cmd+Shift+R) if it looks outdated.
Don't edit files in two places. Pick one location as your "source of truth." We recommend keeping your files on your computer as the main copy, and uploading to GitHub as the publishing step. If you edit files directly on GitHub and on your computer, the two versions will get out of sync and it gets confusing.

Troubleshooting

"I don't see the Go Live button in VS Code"
Make sure you installed the Live Server extension (Step 3 above). Also make sure you have an HTML file open — the Go Live button only appears when you have an HTML file active.
"My CSS isn't loading in the local preview"
Make sure the path in your <link> tag matches your actual file structure. If style.css is in the same folder as index.html, use href="style.css". Open VS Code's built-in terminal (Ctrl+`) and check that both files are in the same folder.
"My page looks different locally vs. on GitHub Pages"
This usually means a file path issue. Check that all your paths are relative (like style.css, not /style.css or C:\Users\...\style.css). What works locally should work on GitHub Pages if you're using relative paths.
"VS Code looks overwhelming"
It's normal to feel that way at first. For this class, you only need to know three things: open a folder (File → Open Folder), edit files in the main editor area, and save with Ctrl+S / Cmd+S. You can ignore everything else.

Quick Checklist

  • I have VS Code installed
  • I installed the Live Server extension
  • I can open my project folder in VS Code
  • I can launch Live Server and see my site in the browser
  • Changes I save in VS Code auto-refresh in the browser
