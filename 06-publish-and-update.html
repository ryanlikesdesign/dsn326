Publish & Update Your Site

How to make changes and see them go live.

← Back to Getting Started

Your Site Is Live — Now What?

Once GitHub Pages is enabled, every change you commit to your repository will automatically update your live website. There's no "publish" button to press — just save (commit) your changes and GitHub rebuilds your site within a minute or two.

How to Edit an Existing File

  1. Go to your repository on GitHub.
  2. Click on the file you want to edit (for example, index.html).
  3. Click the pencil icon (edit button) in the top-right corner of the file view.
  4. Make your changes in the editor.
  5. Scroll down and click "Commit changes".
  6. In the popup, you can write a brief description of what you changed (optional but recommended), then click "Commit changes" again.
Tip: After committing, wait about 1–2 minutes, then visit your site URL and do a hard refresh (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac) to see the updated version. Browsers sometimes cache the old version.

How to Replace a File

If you've been working on a file locally and want to replace the version on GitHub:

  1. Go to your repo and click "Add file" → "Upload files".
  2. Upload the updated file. If it has the exact same name as an existing file, GitHub will replace it.
  3. Commit the changes.

How to Delete a File

  1. Click on the file in your repository.
  2. Click the three-dot menu (⋯) in the top-right corner of the file view.
  3. Select "Delete file".
  4. Commit the deletion.

Checking If Your Site Deployed Successfully

If you want to verify that your latest changes are live:

  1. Go to your repo and click the "Actions" tab.
  2. You'll see a list of recent deployments. A green checkmark means it deployed successfully. An orange circle means it's still building. A red X means something went wrong.
  3. If you see a red X, click on the failed deployment to see what went wrong. The most common issue is a broken file reference (like linking to a CSS file that doesn't exist).

Common Issues & Fixes

"My changes aren't showing up"
Wait 1–2 minutes and do a hard refresh (Ctrl+Shift+R or Cmd+Shift+R). Browser caching is the most common reason changes seem to not work.
"My CSS isn't loading"
Check your <link> tag in the HTML. The href path must exactly match the file location in your repo. If your CSS is at the root, use href="style.css". If it's in a css folder, use href="css/style.css".
"My images aren't showing"
Same idea — check that the src path in your <img> tags exactly matches the file location. Also make sure the image file was actually uploaded to the repo. File names are case-sensitive.
"I get a 404 error"
Make sure your main page is named index.html (not Index.html, not home.html). Also check that GitHub Pages is still enabled in Settings → Pages.

Quick Checklist

  • I know how to edit files directly on GitHub
  • I know how to upload new or replacement files
  • I know to wait 1–2 minutes and hard-refresh after changes
  • I know how to check the Actions tab for deployment status
