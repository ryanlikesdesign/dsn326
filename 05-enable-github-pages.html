DSN 326 — Step 05

Enable GitHub Pages

This is where your repo becomes a live website.

← Back to Getting Started

Before You Start

Make sure you have:

  • A repository with at least an index.html file in it
  • The repository set to Public

Step-by-Step: Turn On GitHub Pages

  1. Go to your repository on GitHub (for example, github.com/yourusername/design-system).
  2. Click the "Settings" tab at the top of the repo page. It's in the navigation bar alongside "Code," "Issues," "Pull requests," etc.
  3. In the left sidebar, scroll down to the "Code and automation" section and click "Pages".
  4. Under "Source", you'll see a dropdown that says "None" or "Deploy from a branch." Select "Deploy from a branch".
  5. A new dropdown will appear asking which branch to deploy. Select main (this is your default branch).
  6. For the folder dropdown, leave it set to / (root). This tells GitHub to look for your index.html at the top level of your repo.
  7. Click "Save".
That's it! GitHub is now building your site. It usually takes 1–3 minutes for your site to go live. You'll see a message at the top of the Pages settings that says "Your site is live at..." with your URL.

Your Site URL

Your published site will be at:
https://yourusername.github.io/your-repo-name/

For example, if your GitHub username is janedoe and your repo is called design-system, your URL will be:
https://janedoe.github.io/design-system/

What If I Don't See My Site?

Wait a couple of minutes. GitHub Pages doesn't publish instantly. Give it 1–3 minutes, then refresh the page. You can also check the "Actions" tab in your repo to see if the deployment is still in progress.
Check that you have an index.html file. GitHub Pages needs a file named exactly index.html (lowercase) at the root of your repo. If the file is named something else or is inside a subfolder, it won't load as the homepage.
Make sure your repo is Public. Go to Settings → General → scroll to "Danger Zone" and check that the repository visibility is "Public."
Check the branch setting. Go back to Settings → Pages and verify the source is set to main branch and / (root) folder.

Quick Checklist

  • I went to Settings → Pages
  • I selected "Deploy from a branch"
  • I chose the main branch and / (root) folder
  • I clicked Save
  • I can see my live site URL
