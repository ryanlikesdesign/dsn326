File Structure & Best Practices

How to organize a multi-page site so everything works correctly.

← Back to Getting Started

Recommended Folder Structure

For a multi-page design system site, organize your repo like this: 

your-repo/
├── index.html          ← Your homepage (required)
├── style.css           ← Your main stylesheet
├── colors.html         ← A page about your color palette
├── typography.html     ← A page about your type system
├── components.html     ← A page about your UI components
└── images/
    ├── logo.png
    ├── color-swatches.png
    └── button-example.png

This is the simplest approach: all HTML pages at the root level, one shared CSS file, and an images folder for any graphics.

Naming Rules

GitHub Pages (and the web in general) are picky about file names. Follow these rules to avoid broken links:

Use all lowercase letters.
colors.html ✓    Colors.html ✗
File names are case-sensitive on the web. If your link says colors.html but the file is named Colors.html, it will break.
Use hyphens instead of spaces.
my-components.html ✓    my components.html ✗
Spaces in file names cause problems in URLs. Use hyphens to separate words.
Avoid special characters.
Stick to letters, numbers, and hyphens. No ampersands, parentheses, or accented characters in file names.

Linking Between Pages

To link from one page to another page in the same folder, just use the file name: 

<a href="colors.html">View Colors</a>
<a href="typography.html">View Typography</a>
<a href="index.html">Back to Home</a>

To link to a file inside a subfolder:

<!-- Linking to an image in the images folder -->
<img src="images/logo.png" alt="Logo">

<!-- Linking to a CSS file in a css folder -->
<link rel="stylesheet" href="css/style.css">
Important: Use relative paths (like style.css or images/logo.png), not absolute paths (like /style.css or C:\Users\me\style.css). Absolute paths won't work correctly on GitHub Pages because your site is hosted inside a subfolder (/your-repo-name/).

Sharing One CSS File Across All Pages

You can (and should) use one CSS file for your entire site. Just include the same <link> tag in the <head> of every HTML page: 

<link rel="stylesheet" href="style.css">

This way, if you change a color or font in style.css, it updates everywhere.

Building a Simple Navigation

To let visitors navigate between your pages, add a <nav> section to every page. Here's a simple example: 

<nav>
  <a href="index.html">Home</a>
  <a href="colors.html">Colors</a>
  <a href="typography.html">Typography</a>
  <a href="components.html">Components</a>
</nav>

Copy and paste this <nav> block into the top of every page's <body> so the navigation is consistent everywhere.

Common Mistakes to Avoid

Don't put index.html inside a subfolder.
Your index.html must be at the root (top level) of your repository, not inside a folder like pages/index.html. GitHub Pages looks for it at the root.
Don't use absolute file paths from your computer.
Paths like C:\Users\jane\Desktop\project\style.css or /Users/jane/style.css will not work on the web. Always use relative paths.
Don't forget to commit after every change.
Your changes aren't saved to GitHub until you click "Commit changes." If you close the tab before committing, your edits are lost.

Quick Checklist

  • My index.html is at the root of the repo
  • All file names are lowercase with hyphens (no spaces)
  • I'm using relative paths for all links and images
  • All pages link to the same style.css file
  • I have a navigation section on every page
Congratulations! You now know everything you need to build and publish a multi-page website using GitHub Pages. If you run into issues, re-check the troubleshooting section in Step 06 or ask your instructor for help.
