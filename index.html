Getting Started with GitHub Pages

Everything you need to host your design system site — no coding experience required.

What You'll Learn

In this course, you'll build a multi-page website for your design system project. To put that site on the internet so anyone can see it, we'll use a free tool called GitHub Pages. This guide walks you through every step — from creating an account to publishing your first page.

Don't worry if you've never heard of GitHub before. By the end of this guide, you'll have a live website at a URL like yourusername.github.io/your-project that you can share with anyone.

The Steps

Follow these pages in order. Each one covers a single topic and includes screenshots and tips.

01

What Is GitHub?

A plain-English explanation of GitHub — what it is, why designers use it, and how it's different from Google Drive or Dropbox.

 02

Create a GitHub Account

Sign up for a free GitHub account. We'll walk through the registration form step by step.

 03

Create a Repository

A repository is like a project folder. You'll create one to hold all of your website files.

 04

Add Your Files

Upload your HTML, CSS, and image files to your repository — right from your browser.

 05

Enable GitHub Pages

Flip the switch that turns your repository into a live website. This is where the magic happens.

 06

Publish & Update Your Site

See your live URL, learn how to make changes, and troubleshoot common issues.

 07

File Structure & Best Practices

How to organize a multi-page site so everything links correctly and loads properly.

 08

Previewing Your Work

How to preview your site on your own computer using VS Code and Live Server — so you can see changes instantly before uploading.

What You'll Need

  • A computer with a web browser (Chrome, Firefox, Safari, or Edge)
  • An email address to create your GitHub account
  • Your HTML and CSS files (we'll create starter files if you don't have any yet)

Good news: You do not need to install any software, use the command line, or know Git. Everything in this guide happens in your web browser.
